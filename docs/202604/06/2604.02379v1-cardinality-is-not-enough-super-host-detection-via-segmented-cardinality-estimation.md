## 摘要
准确检测与大量不同对等端建立连接的超级主机，对于缓解网络攻击和确保高质量的网络服务具有重要意义。现有的基于 Sketch 的方法根据完整的 IP 地址来估计不同连接的数量（称为流基数），但忽略了恶意或受害超级主机通常与同一子网内的主机进行通信这一事实，从而导致高误报率和低准确率。虽然基于层次结构的方法可以捕获子网内的流基数，但它们本质上存在内存占用高的问题。为了解决这些局限性，我们提出了 SegSketch，这是一种分段基数估计方法，它采用轻量级的减半段哈希策略来推断 IP 地址的公共前缀长度，并在受限的内存大小下估计子网内的基数，以提高检测准确率。基于真实世界轨迹的实验表明，与最先进的解决方案相比，SegSketch 将 F1 分数提高了多达 8.04 倍，特别是在内存预算较小的情况下。

## Abstract
Accurately detecting super host that establishes connections to a large number of distinct peers is significant for mitigating web attacks and ensuring high quality of web service. Existing sketch-based approaches estimate the number of distinct connections called flow cardinality according to full IP addresses, while ignoring the fact that a malicious or victim super host often communicates with hosts within the same subnet, resulting in high false positive rates and low accuracy. Though hierarchical-structure based approaches could capture flow cardinality in subnet, they inherently suffer from high memory usage. To address these limitations, we propose SegSketch, a segmented cardinality estimation approach that employs a lightweight halved-segment hashing strategy to infer common prefix lengths of IP addresses, and estimates cardinality within subnet to enhance detection accuracy under constrained memory size. Experiments driven by real-world traces demonstrate that, SegSketch improves F1-Score by up to 8.04x compared to state-of-the-art solutions, particularly under small memory budgets.

在网络安全领域，检测**超级主机**（Super Host，包括向大量目标发送连接的“超级传播者”和接收大量连接的“超级接收者”）对于防御扫描攻击、垃圾邮件分发及 DDoS 攻击至关重要。
*   **现有挑战：** 传统的基于 Sketch（摘要结构）的方法仅关注**流基数**（Flow Cardinality，即不同 IP 的总数）。然而，合法的 Web 服务器或 DNS 解析器同样具有高基数，导致高误报率。
*   **关键观察：** 恶意攻击（如子网扫描或针对特定应用集群的 DDoS）通常具有明显的**子网聚集性**，即连接的 IP 往往共享相同的网络前缀。
*   **研究目标：** 在有限的内存资源下，如何同时识别 IP 的公共前缀并准确估计子网内的基数，从而区分恶意超级主机与合法高基数主机。

#### 2. 方法论：SegSketch 核心设计
论文提出了 **SegSketch**，其核心思想是通过轻量级结构推断 IP 前缀长度并进行分段基数估计。
*   **数据结构：** 采用 $r$ 行 $c$ 列的桶（Bucket）结构。每个桶包含：
    1.  **Host Key**：存储主机的 IP。
    2.  **Subnet Bitmap**：用于推断公共前缀长度。
    3.  **Host Bitmap**：用于估计该子网内的流基数。
*   **关键技术：减半分段哈希（Halved-segment hashing）**
    *   将 IP 地址划分为 $V$ 个段（每段 $G$ 位）。
    *   当新数据包到达时，递归地对 IP 段进行 2-value 哈希。如果某段哈希结果一致，则在位图中选择对应的一半区域；若不一致，则停止递归。通过位图被覆盖的范围，可以推断出该主机连接目标的公共前缀长度。
*   **基数估计：** 利用 **Linear Counting** 算法。根据推断出的前缀长度，仅将 IP 的主机部分哈希进 Host Bitmap，从而实现针对子网的精准计数。
*   **更新与替换策略：** 采用基于概率的替换机制。如果桶已满，新主机以与当前桶内主机基数成反比的概率进行替换，确保高基数主机能稳定驻留。
