## 摘要
微服务中的分布式追踪对于诊断至关重要，但会产生海量数据，因此需要智能采样。为了最大化保真度，最先进的（SOTA）尾部采样器通过将完整的（甚至是日志增强的）追踪建模为图来进行分析。然而，这种对计算开销巨大的图分析的依赖造成了性能瓶颈，阻碍了它们在在线环境中的应用。为此，我们提出了 Gleaner，一个打破了这种权衡的在线尾部采样框架。它的核心洞察是：显式的图结构对于高保真追踪分组并非必要。相反，Gleaner 将每个追踪表示为增强了日志语义的“边集（bag-of-edges）”，用高效的基于集合的操作取代了缓慢的图算法。它还采用了警报驱动的配额和多样性保留策略，为下游的根因分析（RCA）优先处理异常和罕见的追踪。实验表明，Gleaner 处理每个追踪仅需 0.74 毫秒，与基准方法相比，追踪模式覆盖率提升了高达 128.7%，香农熵提升了高达 32.9%。在仅 1% 的采样率下，Gleaner 的 RCA 准确率比次优采样器提高了 42%-107%。此外，在 Gleaner 采样数据上进行的 RCA 比使用完整的、未采样的数据集更准确。这一结果将智能采样从一种数据缩减技术重新定义为一种用于自动化运维的强大信号增强范式。

## Abstract
Distributed tracing in microservices is critical for diagnostics but generates overwhelming data volumes, necessitating intelligent sampling. To maximize fidelity, state-of-the-art (SOTA) tail-based samplers analyze complete (or even log-enriched) traces by modeling them as graphs. However, this reliance on computationally expensive graph analysis creates a performance bottleneck that prohibits their use in online settings.   To this end, we propose Gleaner, an online tail-sampling framework that breaks this trade-off. It is founded on the key insight that explicit graph structures are unnecessary for high-fidelity trace grouping. Instead, Gleaner represents each trace as a "bag-of-edges" augmented with log semantics, replacing slow graph algorithms with highly efficient set-based operations. It also employs an alarm-driven quota and a diversity-preserving strategy to prioritize anomalous and rare traces for downstream Root Cause Analysis (RCA). Experimentally, Gleaner processes traces at 0.74ms each, improving Trace Pattern Coverage by up to 128.7% and Shannon Entropy by up to 32.9% over baselines. At just a 1% sampling rate, Gleaner improves RCA accuracy by 42%-107% over the next-best sampler. Moreover, RCA on Gleaner's sampled data is more accurate than with the entire, unsampled dataset. This result reframes intelligent sampling from a data reduction technique to a powerful signal enhancement paradigm for automated operations.

### 1. 核心问题与研究动机
*   **核心挑战**：微服务系统每天产生数以 TB 计的追踪（Trace）数据，其中绝大多数是重复的正常请求，而关键的异常信号往往隐藏在长尾分布中。
*   **现有技术局限**：
    *   **头部采样（Head-based）**：在请求开始时随机决定是否采样，极易丢失罕见或异常的追踪。
    *   **尾部采样（Tail-based）**：虽能利用完整信息，但现有 SOTA 方法（如基于图神经网络 GNN 或复杂图内核的方法）计算开销巨大，难以满足高吞吐在线环境的实时性要求。
    *   **语义缺失**：大多数采样器仅关注 Span 级别的元数据（如延迟、状态码），忽略了 Span 内部丰富的日志（Log）语义，导致丢失关键诊断线索。
*   **研究目标**：打破“语义丰富度”与“计算效率”之间的权衡，实现一种既能感知日志语义、又能实时处理的高保真采样器。

### 2. 方法论：Gleaner 框架
Gleaner 的核心思想是：**显式的图结构对于高保真采样并非必要，可以用轻量级的集合运算替代。**

*   **追踪编码器（Trace Encoder）**：
    *   **EPS 表示法**：将 Trace 建模为“事件对集合”（Event-Pair Set, EPS）。它将 Span 的生命周期（开始/结束）、内部日志模板、错误状态和性能退化视为事件，通过 Bigram（双字母组）编码捕获调用关系和执行流。
    *   **异常评分**：根据 Span 状态、日志级别（WARN/ERROR）和延迟偏差计算细粒度的异常分数 $A(T)$。
*   **配额分配器（Quota Allocator）**：
    *   **告警驱动**：与外部监控系统集成。当发生告警时，动态增加相关 API 端的采样配额。
    *   **根 Span 分组**：按入口 API 对 Trace 进行分组，确保不同业务路径的可见性，并利用组内高同质性加速后续计算。
*   **DPP 选择器（DPP Selector）**：
    *   **多样性保留**：利用行列式点过程（DPP）在每个分组内选择既具有高异常分又具有结构多样性的 Trace 子集。
    *   **工程优化**：通过“跨批次相似度缓存”和“基于组内相似性的早期停止策略”，将 DPP 的计算复杂度降低到在线处理水平。

*   **诊断增强**：在仅 1% 采样率下，Gleaner 将下游 RCA 工具的准确率提升了 42%~107%。

### 7. 优点与亮点
*   **高效的 EPS 表示**：巧妙地避开了昂贵的图算法，利用哈希集合运算实现了语义感知。
*   **告警集成**：模拟 SRE 的关注点，将采样策略与实时监控状态挂钩，具有极强的实战意义。
*   **开源贡献**：贡献了一个包含 161 个故障案例的大规模多模态数据集，填补了该领域高质量 Benchmark 的空白。

### 8. 不足与局限
*   **表示法有损**：EPS 虽然高效，但丢失了精确的调用层级和时间戳顺序（尽管作者认为这对于采样决策已足够）。

*   **环境差异**：实验主要在开源基准系统上完成，真实工业环境中的日志噪声、追踪上下文断裂等复杂情况可能对效果产生一定影响。
